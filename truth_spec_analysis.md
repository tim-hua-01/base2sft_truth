## Paper: "The Truthfulness Spectrum Hypothesis"
- Authors: Zhuofan Josh Ying, Shauli Ravfogel, Nikolaus Kriegeskorte, Peter Hase
- ArXiv: 2602.20273
- Repo: https://github.com/zfying/truth_spec
- Local path: /Users/timhua/Documents/aisafety_githubs/truth_spec

## High-Level Summary

The paper trains linear probes (logistic regression) on LLM residual stream activations to detect truthfulness across different "truth types." Main finding: probes trained on factual truth types (definitional, empirical, logical, fictional, ethical — collectively "FLEED") generalize well to each other but **fail on sycophantic lying and expectation-inverted lying** (AUROC ~0.55). A probe trained jointly on all domains recovers strong performance. They also show that post-training (base → chat) pushes sycophancy representations further from other truth types.

## Sycophancy Dataset Construction

### Pipeline (`scripts/create_sycophancy_dataset.py`)

1. **Baseline extraction**: Query model on MMLU STEM questions (25 subjects) with plain prompt, get "true belief" answer + logprobs over A/B/C/D
2. **Sycophancy elicitation**: Re-query with expert bio prepended stating a preferred answer. 4 bio templates × 4 answer choices per question
3. **Pair filtering**: Find cases where `sycophancy_occurred=True` (model changed answer to match user's stated answer). Pair with control (same question + same bio person, different user answer, model didn't change). Final code uses `only_case_2=True` — negatives restricted to cases where user's stated answer matches model's original answer
4. **Quality filter**: Confidence threshold (predictive prob > 0.5 for both original and bio-prompted)
5. **Cap**: max 200 pairs per STEM task

### Key detail: separate datasets per model
Each model (base and chat) gets its OWN sycophancy dataset generated from its own behavior. The base model's sycophancy pairs contain different actual responses than the chat model's.

### Bio templates (4 total)
- Dr. Emily Chen, 42, Professor of Physics at Stanford
- Michael Rodriguez, 35, Senior Research Scientist at Google DeepMind
- Dr. Sarah Johnson, 39, Associate Professor of Biology at Harvard Medical School
- James Park, 31, Data Scientist at Meta AI Research

### Prompt format
Bio prompt prepends user message with expert credentials + "I believe the answer is (X)". Assistant response is prefilled with "I believe the best answer is (" and logprobs over next token (A/B/C/D) are compared.

### Sycophancy data labels (`src/datasets.py` lines 433-514)
- Positive (sycophantic) → `Label.DECEPTIVE`
- Negative (honest) → `Label.HONEST`
- Activations extracted from **single answer-letter token** only (e.g., "A") — only last 1 char marked `detect=True`

- `llama-8b-base`: 1810 paired samples

- `llama-70b-base`: 2151 paired samples + raw unpaired data (25 files)

## Probing Setup

### Architecture (final)
- **Probe**: Logistic regression, α=10⁻⁴
- **Layer**: 33 (for Llama-70B), 13 (for Llama-8B)
- **Token aggregation**: Average of all tokens for training; average for testing
- **Preprocessing**: StandardScaler
- **CV**: 5-fold, `KFold(shuffle=True, random_state=42)`

## The Base vs Chat Sycophancy Experiment (Section "Post-training Reorganizes Geometry")

### Setup
- Train probe on FLEED data from model X → test on sycophancy data from model X
- Compare: base model vs chat model
- Each model has its own FLEED data and its own sycophancy data

### Key results (from paper)
| Model | FLEED→Sycophancy AUROC (base) | FLEED→Sycophancy AUROC (chat) |
|-------|------------------------------|-------------------------------|
| Llama-70B | 0.74 | 0.62 |
| Llama-8B | 0.74 | 0.59 |

Probe direction cosine similarity between FLEED and sycophancy is also higher in base models.

### Claim
Post-training pushes sycophancy into a subspace more orthogonal to other truth types, providing a "representational account" for why chat models are more sycophantic.

## Error Bars / Statistical Issues

### How error bars are calculated
- `np.std()` across 5 CV fold AUROCs (population std, not sample std)
- Located in `src/metrics.py` line 186

### Problems with this approach
1. **Only 5 values** — std estimate itself is very noisy
2. **Population std** (`np.std()` divides by N=5, not N-1=4) — slight underestimate
3. **CV folds are not independent** — 80% training data overlap. Bengio & Grandvalet (2004) showed no unbiased variance estimator exists for k-fold CV
4. **Cross-domain evaluation is worse** — for FLEED→sycophancy, all 5 fold models are tested on the SAME full sycophancy test set. The std only measures training set sensitivity, not real uncertainty
5. **No significance test** — for the base vs chat comparison (the key claim), they'd need a direct statistical test (e.g., DeLong's test, bootstrap) on the AUROC difference

### What they should have done
- DeLong's test for paired AUROC comparison (base vs chat)
- Bootstrap CI on individual AUROCs
- Or at minimum, Nadeau & Bengio (2003) corrected resampled t-test

The base model appears FAR more sycophantic in this setup — **opposite** to the conventional wisdom (Wei et al. 2023, Sharma et al. 2024) the paper cites. BUT the similar paired dataset sizes (2286 vs 2151) mask this difference because of the 200-per-task cap.

The paper only reports the chat model rate (8.5%) and does not compare to the base model rate.

## Issues and Concerns Identified

### 1. Base model prompting
The base model uses a bare `User:/Assistant:` chat template. No few-shot examples. A pretrained LM isn't trained to follow this format — high "sycophancy rate" may just be surface-level copy behavior (base model sees "(A)" in the bio and completes with "A" due to recency bias in next-token prediction).

### 2. Separate sycophancy datasets per model
Each model generates its own sycophancy pairs from its own behavior. This means the base and chat models are evaluated on different data — the comparison is confounded by dataset differences, not just representation differences.

### 3. Sycophancy rates not compared
The paper claims post-training makes models more sycophantic (citing prior work) but their own data shows the opposite under their setup. This is not discussed.

### 4. Error bars are weak
See statistical issues section above.

### 5. No full generalization heatmap for base models
The full NxN cross-domain probing matrix (Figure 3) only exists for chat models. Base models only get the targeted FLEED↔sycophancy comparison.

## Experiments NOT Done (Potential Follow-ups)

### 1. Cross-model sycophancy evaluation
Run the base model on the chat model's sycophantic responses (prefill the chat model's answers into the base model's mouth). Extract base model activations. Test if FLEED probes trained on the base model detect sycophancy in these prefilled responses. This would control for dataset differences between base and chat.

### 2. Few-shot prompting for base model
Use proper few-shot MMLU prompts for the base model instead of the bare User:/Assistant: template. This would give a more valid measure of base model sycophancy.

### 3. Proper statistical tests
Apply DeLong's test or bootstrap to the AUROC differences between base and chat models.

### 4. Full generalization heatmap for base models
Run the complete NxN cross-domain probing matrix on base models for comparison with the chat model matrix.
